### DAY 1

**视频编码就是：用尽可能少的 bit，表示尽可能清楚的视频。**

1920 × 1080，30 fps，RGB，每个通道 8 bit

```python
每一帧有 1920 × 1080 个像素
每秒 30 帧
每个像素有 R、G、B 三个通道
每个通道 8 bit
```

那么每秒**原始**数据量大概是：

```
1920 × 1080 × 3 × 8 × 30
≈ 1.49 Gbit/s
≈ 186 MB/s
```

#### 冗余

**空间冗余：一张图内部很像（同一帧图像中，相邻像素通常很相似。）**

**编码器可以利用周围像素去预测当前像素。，当前像素可以由旁边的像素猜出来。**



**时间冗余：相邻帧之间很像（视频前后帧之间通常很相似）**

**当前帧可以从前一帧“搬过来一部分”，再修修补补**



**视觉冗余：人眼看不出来的细节可以少存（有些信息客观存在，但人眼不敏感，可以少存甚至丢掉。）**

**视频编码常用YUV**

```python
Y：亮度
U/V：色度
```



**统计冗余：有些符号出现得特别多**

**常见的东西用短码，不常见的东西用长码。**



**无损压缩:熵编码只是换一种更短的表示方式，不改变数据本身。有损压缩:(量化),有损压缩就是牺牲一点不重要的细节，换来更低的码率。**

最主要的有损步骤是：

```
量化
```

比如原始值是：

```
101, 102, 103, 104
```

量化后可能都变成：

```
100, 100, 100, 100
```

这样数据变简单了，但是细节丢了一点。



**像素 Pixel:像素就是图像的最小单位。**

**帧 Frame:视频就是很多张图像连续播放。每一张图像叫一帧。**

#### **比特深度 Bit Depth**

**比特深度表示每个通道用多少 bit 表示。**

常见的是：

```
8 bit
```

表示一个通道可以表示：

```
0 到 255
```

共 256 个等级。

如果是 RGB，每个像素有三个通道：

```
R：8 bit
G：8 bit
B：8 bit
```

所以每个像素需要：

```
24 bit
```



####  **码率 Bitrate**

****

比如：

```
4 Mbps
```

表示这个视频每秒大约需要：

```
4,000,000 bit
```

码率越高，通常画质越好，但文件越大，传输压力越大。



#### 信息量：**越罕见的事情，信息量越大**

一个消息到底包含多少信息？平均需要多少 bit 来表示？

**越罕见的事情，信息量越大**

所以信息量和概率有关：

```
概率越大，信息量越小
概率越小，信息量越大
```

公式是：

```
I(x) = -log₂ p(x)
```

其中：

```
p(x)：事件 x 出现的概率
I(x)：事件 x 的信息量
```

####  熵：平均每个符号至少需要多少 bit 来表示。

如果一个信源很**容易预测，它的熵就低。**

熵的公式是：

```
H = Σ p(x) （-log₂ p(x)）=（I（x））
H = Σ p(x) （-log₂ p(x)）
```

**熵就是平均每个符号至少需要多少 bit 来表示。**





原始像素值可能很复杂。

**但是经过预测后，我们不直接编码像素，而是编码残差：**

```
残差 = 原始值 - 预测值
```

如果预测很准，**残差大部分接近 0。**

原始像素可能是：

```
102, 105, 108, 110, 115
```

残差可能是：

```
0, 1, 0, -1, 2
```

残差的分布更集中，很多值接近 0。

这意味着：

```
残差的熵更低！！！
```

**视频编码不是随便压，它是在想办法把原本复杂的数据变成更简单、更可预测、更低熵的数据。**



### DAY 2

所以原始视频可以理解为一个四维数据：

```
时间 × 高度 × 宽度 × 通道
```

其中：

比如一个视频张量可以写成：

```
T × H × W × C
```

```
T：帧数
H：图像高度
W：图像宽度
C：通道数
```



#### 视频信号

一张图片记录的是某一时刻的**空间信息。**

视频记录的是：

```
空间信息 + 时间信息
```

所以视频编码必须同时处理两个方向的冗余：

```
空间方向：同一帧内部的相似性
时间方向：相邻帧之间的相似性
```

#### 帧率：一秒有多少帧

帧率就是：

**每秒播放多少张图像。**



#### YUV

**亮度 Y：认真保存 	色度 U/V：适当少保存**

##### 9.1 YUV 4:4:4

4:4:4 表示：

```
Y、U、V 三个通道分辨率一样
```

也就是说，每个像素都有完整的：

```
Y + U + V
```

如果一帧图像是：

```
1920 × 1080
```

那么：

```
Y：1920 × 1080
U：1920 × 1080
V：1920 × 1080
```

数据量最大，质量最好。

**每个像素都完整保存亮度和颜色。**

------

##### 9.2 YUV 4:2:2

4:2:2 表示：

```
Y 保持完整
U/V 水平方向减半
垂直方向不减半
```

如果 Y 是：

```
1920 × 1080
```

那么 U 和 V 大约是：

```
960 × 1080
```

**也就是说，两个相邻像素共享一组色度信息。**

**亮度每个像素都有，颜色横向少存一半。**



#### 9.3 YUV 4:2:0

4:2:0 是视频编码里**最常见**的格式。

它表示：

```
Y 保持完整
U/V 水平方向减半
U/V 垂直方向也减半
```

如果 Y 是：

```
1920 × 1080
```

那么 U 和 V 大约是：

```
960 × 540
```

也就是说，**一个** **2×2 的像素块共享一组 U/V 色度信息。**

可以这样理解：

```
4 个像素各自有 Y
4 个像素共享 U
4 个像素共享 V
```

所以对于 4 个像素：

```
Y 有 4 个
U 有 1 个
V 有 1 个
```

总共是：

```
4 + 1 + 1 = 6 个采样值
```

**而 RGB 或 YUV 4:4:4 对 4 个像素需要：**

```
4 × 3 = 12 个采样值
```

**所以 YUV 4:2:0 的数据量大约是 YUV 4:4:4 的一半。**

大白话：

**亮度完整保存，颜色横向和纵向都少存一半。**



#### 码率 每秒多少 bit。

**原始**视频码率可以粗略写成：

```
码率 = 宽 × 高 × 每像素bit数（bpp） × 帧率
```

**如果是 RGB 8bit：**

```
每像素 bit 数 = 8 × 3 = 24 bit
```

**如果是 YUV 4:2:0 8bit：**

```
平均每像素 bit 数 = 12 bit
```



**文件大小**

```python
文件大小 = 码率 × 时长 ÷ 8   #单位Byte
```



#### bpp 平均每个像素用了多少 bit

**点云压缩**里也有类似指标

比如：

```
bits per point
```

也就是：

```
每个点平均用了多少 bit
```



编码格式规定：

> **视频数据怎么压缩、怎么解码。**
>
> H.264 / AVC
> H.265 / HEVC
> H.266 / VVC
> AV1

封装格式规定：

> **视频、音频、字幕等内容怎么打包在一个文件里。**
>
> MP4
> MKV
> AVI
> MOV

大白话：

```
H.264 是压缩方法
MP4 是文件盒子
```



#### 鲁棒

**鲁棒不是单纯指效果好，而是指“换数据、换场景、加噪声、点云变稀疏或变稠密时，模型仍然不容易失效”。**



### DAY 3

**视频编码的本质，是把原本庞大、连续、复杂的视频信号，变成离散、可预测、低熵、可压缩的数字码流。**

拆开来看：

```
连续信号
   ↓ 采样
离散位置上的样本
   ↓ 量化
有限精度的数字值
   ↓ 去冗余
更容易预测的数据
   ↓ 熵编码
更短的码流
```

#### 采样

在视频里，采样主要有三种：

```
空间采样：一帧图像里取多少像素
时间采样：一秒钟取多少帧
颜色采样：亮度和色度分别取多少
```

空间采样：分辨率就是空间采样密度

 时间采样：帧率就是时间采样密度

 颜色采样：YUV 4:2:0 就是在减少色度采样

#### 量化

**量化会降低数值精度，把很多不同的值合并到同一个等级，并使许多小系数变成 0！！**

**量化就是把连续或高精度的数，变成有限等级的数。**

所以量化就是把连续值**映射**到有限等级。



这里要特别注意。

“量化”在视频里有两个层面的含义。

##### 第一种：数字化量化

这是把真实世界的连续亮度变成数字。

比如：

```
真实亮度 → 8 bit 像素值 0~255
```

这属于视频采集阶段。

例如摄像头拍摄时，把模拟信号变成数字图像。

##### 第二种：编码压缩量化

这是视频编码时，为了进一步压缩，把变换系数变粗。

比如：

```
DCT 系数 23.7 → 量化后变成 2
DCT 系数 3.1 → 量化后变成 0
```

这属于视频编码阶段。

后面第 5 章“量化与码率控制”重点讲的是第二种。

先记住：

> **数字化量化是为了让计算机能表示视频；编码量化是为了减少码率。**

#### 量化步长

量化步长可以理解为：

**数值被分组时，每一组的宽度。**

比如量化步长是 10：

```
0~9      → 0
10~19    → 1
20~29    → 2
30~39    → 3
```

量化步长越大，分得越粗。

#### 信息量

信息量的大白话解释是：

**一件事越意外，它带来的信息量越大。**

所以信息量和概率有关：

```
概率越大 → 信息量越小
概率越小 → 信息量越大
```

公式是：

```
I(x) = -log₂ p(x)
```

其中：

```
p(x)：事件 x 出现的概率
I(x)：事件 x 的信息量
```

只要记住：

> **越少见，越难猜，信息量越大，需要的 bit 越多。**

**这和压缩非常相关。**

**如果某个符号很常见，应该用短码。如果某个符号很罕见，可以用长码。**

**这就是熵编码的直觉。**

#### 熵:平均每个符号至少需要多少 bit

熵可以理解为：

> **一个信源平均有多难预测。**

熵的公式是：

```
H = Σ p(x) * -log₂ p(x)
```

大白话：

**熵就是平均每个符号至少需要多少 bit 来表示。**



**预测为什么能降低熵？**

**预测编码不是直接压原始值，而是先猜，再只压猜错的部分。猜得越准，残差越小，熵越低，码率越低。**



**量化为什么也能降低熵？**

**量化会把很多不同的值合并成同一个值。**

比如原来残差是：

```
-2, -1, 0, 1, 2, 3
```

量化后可能变成：

```
0, 0, 0, 0, 0, 1
```

这样符号种类减少了，而且 0 出现频率变高。

**这会让分布更集中，熵降低。**

#### 视频编码的典型流程

```
预测 → 残差 → 变换 → 量化 → 熵编码
```



#### 冗余和熵的关系

昨天我们学过冗余。今天要把冗余和熵联系起来。

**冗余可以理解成：**

**数据中可以被预测、可以被省略、可以被更短表示的部分。**

如果冗余多，说明数据有规律。

有规律就意味着：

```
熵低
```

如果数据完全随机，没有规律，那冗余少。

冗余少就意味着：

```
熵高
```

**所以视频编码的目标就是：**

```
利用冗余 → 降低熵 → 减少码率
```



**预测降低数据幅度**
**量化降低数据精度**
**熵编码降低表示长度**



####  采样、量化、熵编码之间的关系

| 概念   | 大白话              | 是否有损 | 举例                |
| ------ | ------------------- | -------- | ------------------- |
| 采样   | 选哪些位置/时刻记录 | 可能有损 | 1080P、30fps、4:2:0 |
| 量化   | 每个值保留多精细    | 通常有损 | 8 bit、QP、Qstep    |
| 熵编码 | 用更短码字表示符号  | 无损     | Huffman、算术编码   |

注意：

```
采样和量化通常会改变信息本身
熵编码只改变表示方式，不改变信息本身
```
