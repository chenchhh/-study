### DAY 1

**视频编码就是：用尽可能少的 bit，表示尽可能清楚的视频。**

1920 × 1080，30 fps，RGB，每个通道 8 bit

```python
每一帧有 1920 × 1080 个像素
每秒 30 帧
每个像素有 R、G、B 三个通道
每个通道 8 bit
```

那么每秒**原始**数据量大概是：

```
1920 × 1080 × 3 × 8 × 30
≈ 1.49 Gbit/s
≈ 186 MB/s
```

#### 冗余

**空间冗余：一张图内部很像（同一帧图像中，相邻像素通常很相似。）**

**编码器可以利用周围像素去预测当前像素。，当前像素可以由旁边的像素猜出来。**



**时间冗余：相邻帧之间很像（视频前后帧之间通常很相似）**

**当前帧可以从前一帧“搬过来一部分”，再修修补补**



**视觉冗余：人眼看不出来的细节可以少存（有些信息客观存在，但人眼不敏感，可以少存甚至丢掉。）**

**视频编码常用YUV**

```python
Y：亮度
U/V：色度
```



**统计冗余：有些符号出现得特别多**

**常见的东西用短码，不常见的东西用长码。**



**无损压缩:熵编码只是换一种更短的表示方式，不改变数据本身。有损压缩:(量化),有损压缩就是牺牲一点不重要的细节，换来更低的码率。**

最主要的有损步骤是：

```
量化
```

比如原始值是：

```
101, 102, 103, 104
```

量化后可能都变成：

```
100, 100, 100, 100
```

这样数据变简单了，但是细节丢了一点。



**像素 Pixel:像素就是图像的最小单位。**

**帧 Frame:视频就是很多张图像连续播放。每一张图像叫一帧。**

#### **比特深度 Bit Depth**

比特深度表示**每个通道用多少 bit 表示。**

常见的是：

```
8 bit
```

表示一个通道可以表示：

```
0 到 255
```

共 256 个等级。

如果是 RGB，每个像素有三个通道：

```
R：8 bit
G：8 bit
B：8 bit
```

所以每个像素需要：

```
24 bit
```



####  **码率 Bitrate**

**单位时间内视频需要多少 bit。**

比如：

```
4 Mbps
```

表示这个视频每秒大约需要：

```
4,000,000 bit
```

码率越高，通常画质越好，但文件越大，传输压力越大。



#### 信息量：**越罕见的事情，信息量越大**

一个消息到底包含多少信息？平均需要多少 bit 来表示？

**越罕见的事情，信息量越大**

所以信息量和概率有关：

```
概率越大，信息量越小
概率越小，信息量越大
```

公式是：

```
I(x) = -log₂ p(x)
```

其中：

```
p(x)：事件 x 出现的概率
I(x)：事件 x 的信息量
```

####  熵：平均每个符号至少需要多少 bit 来表示。

如果一个信源很**容易预测，它的熵就低。**

熵的公式是：

```
H = Σ p(x) （-log₂ p(x)）=（I（x））
H = Σ p(x) （-log₂ p(x)）
```

**熵就是平均每个符号至少需要多少 bit 来表示。**





原始像素值可能很复杂。

**但是经过预测后，我们不直接编码像素，而是编码残差：**

```
残差 = 原始值 - 预测值
```

如果预测很准，**残差大部分接近 0。**

原始像素可能是：

```
102, 105, 108, 110, 115
```

残差可能是：

```
0, 1, 0, -1, 2
```

残差的分布更集中，很多值接近 0。

这意味着：

```
残差的熵更低！！！
```

**视频编码不是随便压，它是在想办法把原本复杂的数据变成更简单、更可预测、更低熵的数据。**
